## 一、两种常见提示词风格的痛点

在与AI对话的实践中,主要存在两种常见的提示词使用风格,但它们各自都有明显的痛点:

### 1. "一次对话就要结果"风格的痛点

这是最常见的使用方式:将所有需求一次性写入提示词,期望AI立即生成满意的结果。但现实往往事与愿违:

**问题根源:**
- 人类用户很难在第一次就把需求描述得全面周到
- AI容易误解用户的真实意图
- 只有在AI生成内容后,用户才会发现哪些信息没提供、哪些需求被误解了

**带来的后果:**
- **反复试错**:不断修改提示词,重新生成内容,陷入"修改-生成-再修改"的循环
- **质量妥协**:最终往往只能接受60-70分的差强人意结果
- **成本浪费**:多次生成低质量内容,浪费大量时间和token成本

**典型场景:**
想让AI帮忙起草高考志愿填报建议,第一次提交后发现AI推荐的专业太笼统,没有结合你的具体情况;修改提示词后再次生成,又发现没有考虑就业前景;再改再生成,发现缺少个人兴趣的匹配...如此反复,既耗时又耗钱。

### 2. "提示词链多轮对话"风格的痛点

有经验的用户会预先准备多个提示词,按照"需求澄清→方案推荐→规划→执行→评审"的5个阶段,依次提交给AI,实现类似"暂停并澄清"的效果。

**看似完美,实则:**
- **操作繁琐**:需要手动将预先准备好的提示词逐个复制粘贴到输入区
- **体验不佳**:频繁切换文档和AI聊天窗口,容易打断思路
- **准备工作量大**:需要预先为5个阶段分别编写独立的提示词

**典型场景:**
准备了5个分阶段的提示词文档,在与AI对话时不断切换窗口、复制粘贴,每次还要检查是否粘贴对了,整个过程体验糟糕,反而降低了效率。

## 二、"暂停并澄清提示词"如何破局

### 1. 核心理念

"暂停并澄清提示词(pause-and-clarify-prompts)"是一个开源的提示词工程项目,它的核心思想是:**让AI在生成内容前主动暂停,与人类用户充分澄清需求,然后一次性生成95分质量的内容。**

### 2. 如何解决"一次对话就要结果"的痛点

**问题:**人类很难一次性把需求说清楚,AI容易误解意图。

**解决方案:**
- 不要求用户一次性说清所有需求
- AI进入**RESEARCH模式**,主动提出澄清问题
- 在**INNOVATE模式**推荐多种内容组织方案供选择
- 在**PLAN模式**制定详细的内容生成计划
- 经用户确认后,在**EXECUTE模式**一次性生成高质量内容

**效果对比:**
- **传统方式**:用户描述→AI生成(60分)→修改描述→再生成(70分)→再修改→再生成(75分)...反复多次
- **暂停并澄清**:用户初步描述→AI澄清(RESEARCH)→推荐方案(INNOVATE)→制定计划(PLAN)→一次生成(95分)

### 3. 如何解决"提示词链多轮对话"的痛点

**问题:**需要预先准备5个阶段的独立提示词,手动复制粘贴,操作繁琐。

**解决方案:**
- 使用**统一的"暂停并澄清提示词"模板**,而非5个独立提示词
- 只需在初始提示词中包含RIPER-5系统提示词
- AI自动在响应中提示下一步操作指令
- 用户只需输入简单的模式切换指令(如"ENTER INNOVATE MODE")

**效果对比:**
- **传统方式**:准备5个独立提示词→切换窗口→复制粘贴第1个→AI响应→切换窗口→复制粘贴第2个...
- **暂停并澄清**:准备1个统一提示词→AI响应并提示下一步→输入"ENTER INNOVATE MODE"→AI响应并提示下一步...

### 4. 核心优势总结

| 对比维度 | 一次对话就要结果 | 提示词链多轮对话 | 暂停并澄清提示词 |
|---------|----------------|----------------|----------------|
| **准备工作** | 需精心编写完整提示词 | 需准备5个独立提示词 | 准备1个统一模板 |
| **操作复杂度** | 简单(一次提交) | 繁琐(多次复制粘贴) | 简单(输入指令即可) |
| **需求澄清** | 无,易误解 | 有,但需预先设计 | AI主动澄清 |
| **方案选择** | 无,AI自行决定 | 有,但需预先准备 | AI推荐多个方案 |
| **内容质量** | 60-75分 | 80-90分 | 95分 |
| **修改次数** | 3-5次或更多 | 1-2次 | 通常0次 |
| **Token成本** | 高(多次重新生成) | 中 | 低(一次生成) |
| **用户体验** | 差(反复试错) | 一般(操作繁琐) | 好(流畅高效) |

### 2. 准备"暂停并澄清"风格提示词

根据"暂停并澄清提示词"模板,完整的提示词由4部分组成:

#### 2.1 开场提示词(个性化)

这部分包含你的个人信息,需要根据实际情况填写:

```
我是【2025年刚完成高考的高中生】,想让AI帮我【生成高考志愿专业填报建议】,以便【让我通过你的分析能从推荐的专业中做出明智选择】。

对话流程:我和AI将依次经历4个模式,需明确输入指令才能进入下一模式。

1. RESEARCH模式:AI澄清我的诉求
   输入"ENTER INNOVATE MODE"进入下一模式,否则继续澄清诉求

2. INNOVATE模式:AI推荐若干内容组织方案
   输入"ENTER PLAN MODE"进入下一模式,否则继续讨论方案

3. PLAN模式:AI根据澄清的诉求和选定的方案制定内容生成计划
   输入"ENTER EXECUTE MODE"进入下一模式

4. EXECUTE模式:AI按计划生成内容
   输入"ENTER REVIEW MODE"进入REVIEW模式(AI对比EXECUTE和PLAN是否有偏差)

如果满意EXECUTE模式生成的内容,可跳过REVIEW模式。

使用方法:
复制下方【】内的内容,补充完整其中嵌套的【】内容后粘贴到提示词输入区,发送给AI开始对话。

【
参加高考所在城市:【北京】
三门选考科目:【生物、化学、历史】
高考分数:【589】
擅长科目:【历史、语文】
兴趣爱好:【手机与电脑游戏】
期望的工作类型:【有趣、同事友善、离家近】
】
```

**说明:**将【】中的内容替换为你自己的实际情况。

#### 2.2 开场提示词(通用)

这部分定义AI的角色、行为、输出格式和需要注意的顾虑:

```
角色:【你是一位专业且和蔼的高考志愿填报辅导老师。】

行为:【
根据我的情况、个人兴趣以及相关就业前景和发展潜力数据,为我推荐3~5个最适合的专业。每个专业需包含以下分析:
1.1 核心学习内容
1.2 为什么适合我
1.3 未来职业方向
1.4 就业前景与挑战
1.5 给我的建议
将推荐的专业按适合程度排序(最适合的排在最前),并说明推荐理由。
】

输出:【请用Markdown格式输出志愿专业填报分析和推荐。】

顾虑:【我担心推荐的专业不够具体或缺乏吸引力。】
```

#### 2.3 文本知识库

由于DeepSeek官网聊天应用开启"搜索"功能后无法同时上传文件,项目中的3个PDF知识库文件已转换为纯文本嵌入到完整提示词中(由于篇幅过长,此处略去,完整版本可参考项目中的`full-prompt-for-deepseek.md`文件)。

如果你不需要特定的知识库,可以省略这部分,让AI通过搜索功能获取相关信息。

#### 2.4 RIPER-5系统提示词

这是"暂停并澄清"的核心机制,定义了5个模式的工作流程:

- **RESEARCH模式**:AI只收集信息,提出澄清问题,不做任何建议
- **INNOVATE模式**:AI头脑风暴,推荐多个内容组织方案
- **PLAN模式**:AI制定详细的内容生成计划
- **EXECUTE模式**:AI严格按照计划生成内容
- **REVIEW模式**:AI验证生成的内容是否与计划一致

完整的RIPER-5系统提示词较长,包含每个模式的详细规则和转换机制(详见项目README)。

#### 步骤4:体验RESEARCH模式(需求澄清)

AI会自动进入RESEARCH模式,开始提出澄清问题,例如:

```
   - 是喜欢游戏的视觉设计和美术风格?

```

**重要提示:**不要急于进入下一模式!即使AI声称已完全理解你的需求,也要追问一句:**"在进入下一模式前,你是否还有我的诉求方面的疑问?我可以回答。"**

```
**方案三:就业导向型**

我个人推荐**方案三(就业导向型为主,兼顾兴趣)**,理由是:...

您更倾向于哪种方案?或者希望我调整推荐方向?

下一步:
1. 进入下一模式的指令 'ENTER PLAN MODE'
2. 继续停留在本模式讨论...
```

输入"ENTER PLAN MODE"后,AI会制定详细的内容生成计划:

```
4. 按适合程度排序并说明理由
5. 生成Markdown格式的完整报告

下一步:
1. 进入下一模式的指令 'ENTER EXECUTE MODE'
2. 继续停留在本模式讨论...
```

### 4. 实战效果对比

让我们对比一下使用"暂停并澄清提示词"前后的效果:

#### 使用前(传统"一次对话就要结果"方式):

- **第1次尝试**:提示词写得不够详细→AI推荐的专业太笼统,没有结合个人情况→质量60分
- **第2次尝试**:补充个人信息→AI推荐的专业没有考虑就业前景→质量70分
- **第3次尝试**:强调就业前景→AI推荐的专业没有结合兴趣爱好→质量75分
- **第4次尝试**:再次修改提示词→AI推荐的专业分析深度不够→质量80分
- **最终**:试了4-5次,勉强接受80分的结果,但仍不够满意

**成本:**
- 时间成本:编写和修改提示词累计30-60分钟
- Token成本:生成了4-5次完整内容,消耗大量tokens
- 心理成本:反复试错,挫败感强

#### 使用后("暂停并澄清提示词"方式):

- **RESEARCH模式**:AI主动澄清需求,补充了很多我没想到的信息(5-10分钟)
- **INNOVATE模式**:AI推荐了3种方案,我选择了最符合期望的(3-5分钟)
- **PLAN模式**:AI制定详细计划,确保生成内容全面(2-3分钟)
- **EXECUTE模式**:AI一次性生成高质量内容,质量95分!(一次生成)

**成本:**
- 时间成本:累计20-30分钟(包含澄清时间)
- Token成本:只生成1次完整内容,节省60-80%
- 心理成本:流程清晰,体验流畅,满意度高

## 四、进阶技巧与注意事项

### 1. RESEARCH模式的关键:不要急于进入下一模式

很多用户在RESEARCH模式会犯一个错误:AI提了几个问题,回答后就急于输入"ENTER INNOVATE MODE"进入下一模式。

**正确做法:**
- 即使AI声称"已经完全理解您的需求",也要追问:**"在进入下一模式前,你是否还有我的诉求方面的疑问?我可以回答。"**
- 如果AI已经自行进入INNOVATE模式,要明确要求它:**"请回到RESEARCH模式,我还有需求需要澄清。"**
- 只有当AI明确表示"没有更多问题"时,再进入下一模式

**原因:**充分的需求澄清是高质量内容的基础。前期多花5-10分钟澄清,后期能节省数倍的修改时间。

### 2. INNOVATE模式的关键:充分讨论方案

不要急于接受AI推荐的第一个方案。如果对方案有疑虑,可以:
- 要求AI解释为什么推荐这个方案
- 要求AI提供更多方案选择
- 提出自己的想法,让AI评估可行性

### 3. PLAN模式的关键:确保计划详细

检查AI制定的计划是否包含:
- 明确的工作步骤
- 清晰的输出格式
- 足够的分析深度

如果计划不够详细,要求AI补充完善。

### 5. 成本优化建议

- RESEARCH和INNOVATE模式可以使用较便宜的模型(如GPT-3.5、文心一言等)进行需求澄清
- EXECUTE模式使用更强大的模型(如DeepSeek-V3、Claude-3.5-Sonnet等)确保内容质量
- 开启DeepThink功能时token消耗会增加,但内容质量也会显著提升,综合性价比更高
